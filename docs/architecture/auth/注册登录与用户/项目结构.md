后端技术框架为Java17+springboot3+mybatis-plus+PostgreSQL+Redis+Lombok+Knife4j +slf4j+docker

建议的划分方式如下：
1. Auth 模块（身份中心）
   职责：提供“主体（Subject）”的完整画像。
   管什么：用户账号、密码、部门归属、全局角色（如“集团管理员”）、岗位。
   输出什么：当用户请求过来时，Auth 模块（或网关）告诉业务模块：
   “当前用户是 ID=1001，部门=研发部，角色=部门经理，岗位=Java开发”。
2. Folder/File 模块（资源与策略中心）
   职责：定义“客体（Object）”与“策略（Policy）”，并执行判定。
   管什么：文件夹结构、文件元数据、资源授权表（Resource Permission Table）。
   做什么：
   存储：在 folder 数据库中维护一张 permission 表。
   逻辑：处理权限继承（Inheritance）、阻断（Override）、有效期（TTL）。
   判定：拿着 Auth 传过来的“用户画像”，对比自己的“资源授权表”，算出结果。
